## Главный промпт

```
Я разрабатываю Flutter приложение для пассажиров междугороднего такси.

## ВАЖНО: UI-FIRST подход

Сейчас мы делаем ТОЛЬКО ВЁРСТКУ (UI) без логики:
- Все экраны с mock данными
- Без API интеграции
- Без state management
- После ревью UI - будем добавлять логику
- UI должен быть на английском языке (рынок США)

## Целевой рынок: США

- Валюта: USD ($)
- Телефон: +1 (XXX) XXX-XXXX
- Расстояние: miles
- Дата: MM/DD/YYYY
- Время: 12-hour (AM/PM)
- Города: NYC, Boston, LA, San Diego, Chicago, Miami

## Документация (ПРОЧИТАЙ СНАЧАЛА)

1. `docs/UI_UX_PLAN.md` - wireframes всех экранов
2. `docs/DESIGN_SYSTEM.md` - цвета (Teal), шрифты, компоненты
3. `docs/PROJECT_ROADMAP.md` - текущий статус и план

## Правила

1. Следуй дизайн-системе строго (основной цвет - Teal #009688)
2. Используй mock данные с американскими именами/адресами
3. НЕ указывай версии пакетов - устанавливай через `flutter pub add`
4. Пиши чистый Dart код
5. После задачи обнови PROJECT_ROADMAP.md (отметь ✅)

## Текущий этап

[УКАЖИ ФАЗУ из PROJECT_ROADMAP.md]

## Задача

[ОПИШИ КОНКРЕТНУЮ ЗАДАЧУ]
```

## Примеры задач по фазам

### Фаза 1.1: Подготовка
```
Текущий этап: Фаза 1.1 - Подготовка UI

Задача:
1. Создай структуру папок согласно PROJECT_ROADMAP.md
2. Установи UI пакеты (без версий, через flutter pub add)
3. Реализуй AppTheme из DESIGN_SYSTEM.md (светлая + тёмная) - цвет Teal
4. Создай базовые компоненты: AppButton, AppTextField, AppCard
```

### Фаза 1.2: Онбординг и Авторизация
```
Текущий этап: Фаза 1.2 - Онбординг и Авторизация

Задача: Сверстай экраны авторизации согласно UI_UX_PLAN.md:
1. SplashScreen (логотип "Travel Together" + loader)
2. OnboardingScreen (3 слайда с PageView для пассажиров)
3. PhoneInputScreen (ввод телефона US формат +1)
4. OtpScreen (6-значный код с таймером)

Используй mock данные, без реальной логики.
```

### Фаза 1.3: Регистрация пассажира
```
Текущий этап: Фаза 1.3 - Регистрация пассажира

Задача: Сверстай простую регистрацию:
1. ProfileSetupScreen - имя, email (опционально), фото

Простая форма без шагов (пассажиру не нужна верификация).
```

### Фаза 1.4: Главная и Поиск
```
Текущий этап: Фаза 1.4 - Главная и Поиск

Задача: Сверстай главные экраны:
1. BottomNavShell (4 вкладки: Search, Trips, Chats, Profile)
2. HomeScreen (форма поиска + популярные маршруты)
3. SearchResultsScreen (список поездок с фильтрами)
4. FiltersScreen (фильтры: время, цена, рейтинг)

Mock данные: NYC→Boston, LA→San Diego и т.д.
```

### Фаза 1.5: Бронирование
```
Текущий этап: Фаза 1.5 - Бронирование

Задача: Сверстай экраны бронирования:
1. TripDetailsScreen (детали поездки + водитель)
2. BookingScreen (выбор мест, заметка, итого)
3. BookingConfirmationScreen (успешная заявка)

Цены в USD, расстояние в miles.
```

### Фаза 1.6: Мои поездки
```
Текущий этап: Фаза 1.6 - Мои поездки

Задача: Сверстай экраны поездок пассажира:
1. MyTripsScreen (активные + прошлые, с табами)
2. TripStatusScreen (статус забронированной поездки)
3. TripCompletedScreen (оценка водителя)

Mock статусы: awaiting, confirmed, completed.
```

### Фаза 1.7: Чат
```
Текущий этап: Фаза 1.7 - Чат

Задача: Сверстай экраны чата:
1. ChatsListScreen (список диалогов с водителями)
2. ChatScreen (переписка с водителем)

Mock сообщения на английском.
```

### Фаза 1.8: Профиль
```
Текущий этап: Фаза 1.8 - Профиль

Задача: Сверстай экраны профиля:
1. ProfileScreen (меню профиля)
2. EditProfileScreen (редактирование данных)
3. SettingsScreen (уведомления, тема, язык)
4. FavoritesScreen (сохранённые маршруты)

Mock пользователь: John Smith, +1 (555) 123-4567
```

## Отличия от Driver App

| Аспект | Driver | Passenger |
|--------|--------|-----------|
| Цвет | Blue (#1E88E5) | **Teal (#009688)** |
| Регистрация | Сложная (3 шага + документы) | **Простая (1 шаг)** |
| Главный экран | Создание поездок | **Поиск поездок** |
| Навигация | 5 вкладок | **4 вкладки** |
| Верификация | Требуется | **Не требуется** |
